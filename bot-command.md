# DogeBot 机器人命令

本文档集中记录飞书机器人支持的文字命令。命令在机器人收到文本消息时解析；如果消息未命中任何命令，会执行该机器人配置的默认兜底指令，未配置兜底时会原样回复消息内容。

## 命令总览

| 命令 | 用途 | 示例 |
| --- | --- | --- |
| `/users` | 记录和查询当前发起人的 at 用户列表 | `/users @Alice @Bob` |
| `/douyin` | 从当前机器人绑定用户的抖音收藏记录中随机返回一个视频链接 | `/douyin 123` |
| `/set-default` | 设置当前机器人的未命中兜底指令 | `/set-default "/douyin 123"` |
| `/add-cron` | 为当前聊天会话添加定时执行任务 | `/add-cron "*/5 * * * *" "/douyin 123"` |

## `/users`

用户向 bot 发送 `/users` 命令时，服务端会按 `bot_id + at_by + at_who` 记录关系。返回内容是飞书消息卡片，卡片里会把该发起人在当前 bot 下历史记录的 `at_who` 全部 at 出来。

### 用法

```text
/users
/users @Alice @Bob
/users delete
/users delete @Alice
/users top @Alice
/users new 3
```

### 参数说明

- `/users @Alice @Bob`：记录当前发起人 at 过 Alice 和 Bob，并返回当前列表。
- `/users`：不新增记录，只返回当前发起人的历史 at 列表。
- `/users delete`：软删除当前发起人在当前 bot 下的全部历史 at 记录。
- `/users delete @Alice`：只软删除 Alice。
- `/users top @Alice`：记录 Alice，并把 Alice 排到返回卡片的最前面。
- `/users new 3`：只返回最新加入的 3 个 `at_who`。

飞书卡片中的 at 用户格式为：

```text
<at id=ou_xxx></at>
```

## `/douyin`

从当前机器人绑定用户的抖音收藏记录中，按“模拟点击文案”随机选一个 `aweme_id`，并回复抖音视频链接。

### 用法

```text
/douyin {模拟点击文案}
```

### 示例

```text
/douyin 123
```

如果命中记录，机器人回复：

```text
https://www.douyin.com/video/{aweme_id}
```

### 注意事项

- 当前机器人必须绑定到 DogeBot 用户。
- 对应 `{模拟点击文案}` 必须已经由桌面端抖音收藏监听采集并同步到服务端。
- 如果没有记录，会回复 `暂无“{模拟点击文案}”的抖音收藏记录`。

## `/set-default`

设置当前机器人的默认兜底指令。以后给机器人发送未命中任何命令的文本时，会自动执行这个兜底指令。

### 用法

```text
/set-default "{兜底指令}"
```

### 示例

```text
/set-default "/douyin 123"
```

设置后，如果用户给该机器人发送普通文本且没有命中任何命令，服务端会按 `/douyin 123` 执行。

### 注意事项

- 兜底指令按机器人维度保存，每个机器人互相独立。
- 兜底执行时不会再次触发 `/set-default`，避免循环或误改配置。
- 删除机器人时会同步删除该机器人的兜底指令配置。

## `/add-cron`

为当前聊天会话添加定时任务。任务会绑定当前消息所在的 `chat_id`，所以在群里添加就定时发到该群，在单聊里添加就绑定单聊会话。

### 用法

```text
/add-cron "{cron表达式}" "{要执行的指令}"
```

也支持不加引号的 5 段 cron 表达式：

```text
/add-cron */5 * * * * /douyin 123
```

推荐对 cron 表达式和指令都加引号，避免空格解析歧义。

### 示例

每 5 分钟在当前会话执行一次 `/douyin 123`：

```text
/add-cron "*/5 * * * *" "/douyin 123"
```

每天 10:30 在当前会话执行一次 `/douyin 123`：

```text
/add-cron "30 10 * * *" "/douyin 123"
```

### Cron 格式

当前支持 5 段 cron：

```text
分 时 日 月 周
```

字段范围：

| 字段 | 范围 |
| --- | --- |
| 分 | `0-59` |
| 时 | `0-23` |
| 日 | `1-31` |
| 月 | `1-12` |
| 周 | `0-7`，其中 `0` 和 `7` 都表示周日 |

字段语法：

- `*`：任意值。
- `*/n`：每 n 个单位执行一次，例如 `*/5`。
- `a,b,c`：枚举多个值，例如 `1,15,30`。
- `a-b`：范围，例如 `9-18`。
- `a-b/n`：范围内按步长匹配，例如 `9-18/3`。

### 注意事项

- 调度器随服务端启动，默认每 `30s` 扫描一次到期任务。
- 执行时间按服务端本机时间计算。
- 当前定时任务执行只支持 `/douyin {模拟点击文案}`。
- 如果定时任务里的指令不是 `/douyin`，机器人会在目标会话提示该定时任务暂不支持该指令。
- 删除机器人时会同步删除该机器人相关的定时任务。
